Prompt: "Crea la estructura de carpetas para un proyecto Java de inventario 
con arquitectura por capas. Las carpetas dentro de src deben ser:
com/inventario/entidades
com/inventario/dao
com/inventario/servicio
com/inventario/presentacion

También crea estas carpetas en la raíz:
datos/ (con un archivo productos.txt vacío)

README.md

prompts.txt

No escribas código Java todavía, solo la estructura."

PROMPT #4
Usado para: Implementar la persistencia en archivo de texto.
Prompt: "Crea la clase ProductoDAOArchivo.java en el paquete com.inventario.dao
que implemente la interfaz IProductoDAO.

Requisitos:
- Lee y escribe en el archivo \"datos/productos.txt\"
- Cada línea del archivo representa un producto con este formato:
  id|nombre|cantidad|precio
  Ejemplo: 1|Laptop|5|1200.00

- Implementa los 5 métodos: guardar, listarTodos, buscarPorId, 
  actualizar y eliminar

- Para eliminar: reescribe el archivo sin la línea del producto eliminado
- Para actualizar: reescribe el archivo reemplazando la línea del producto
- Maneja excepciones IOException con try-catch, imprime el error en consola

No uses base de datos, solo el archivo .txt"
Modificaciones manuales: Se pudo cambiar manualmente el texto de los mensajes de error o el formato visual de algunos bloques `try-catch`.

PROMPT #6
Usado para: Implementar la capa de servicio con validaciones.
Prompt: "Crea la clase ProductoServicio.java en el paquete com.inventario.servicio
que implemente IProductoServicio.

Requisitos:
- Tiene un atributo privado de tipo IProductoDAO llamado dao
- El constructor recibe un IProductoDAO y lo asigna al atributo
- Implementa los 5 métodos delegando al dao

Validaciones obligatorias en registrarProducto:
- El nombre no puede ser null ni vacío → lanza IllegalArgumentException
- La cantidad no puede ser negativa → lanza IllegalArgumentException  
- El precio debe ser mayor a 0 → lanza IllegalArgumentException
- El id no puede ya existir (usa buscarPorId para verificar) 
  → lanza IllegalArgumentException con mensaje \"ID ya existe\"

En los demás métodos no hace falta validación extra por ahora."
Modificaciones manuales: Se pudo ajustar manualmente el texto de las excepciones o simplificar el orden de las validaciones.

PROMPT #7
Usado para: Crear el menú de consola de la capa de presentación.
Prompt: "Crea la clase Menu.java en el paquete com.inventario.presentacion

Requisitos:
- Tiene un atributo privado IProductoServicio servicio y un Scanner
- El constructor recibe un IProductoServicio
- Método público void iniciar() que muestra un menú en bucle con estas opciones:
  1. Registrar producto
  2. Listar productos
  3. Buscar producto por ID
  4. Actualizar producto
  5. Eliminar producto
  0. Salir

- Cada opción llama a un método privado separado:
  registrarProducto(), listarProductos(), buscarProducto(), 
  actualizarProducto(), eliminarProducto()

- En registrarProducto: pide id, nombre, cantidad y precio al usuario
- En listarProductos: imprime todos los productos en consola
- En buscarProducto: pide un ID e imprime el producto o \"No encontrado\"
- En actualizarProducto: pide el ID y los nuevos datos
- En eliminarProducto: pide el ID, confirma con \"¿Está seguro? (s/n)\" 
  y elimina si confirma

- Captura IllegalArgumentException y muestra el mensaje de error
- No importa ni usa nada del paquete dao directamente

Debe de ser un menu moderno y sencillo de entender"
Modificaciones manuales: Se pudo cambiar manualmente el texto de las opciones, los separadores visuales o la redacción de mensajes al usuario.
